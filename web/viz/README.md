# Purpose
Define the Web UI viz layer.

# What goes here
- Front-end viz files used to present run artifacts and control workflows.
- Client logic that consumes FastAPI responses.

# What does NOT go here
- FastAPI backend code or Python engine logic.
- Raw/processed/artifact data files.

# How it is used
- Reads from API route groups for runs, world drafts, and training as applicable.
- Displays world, fields, and metrics generated by monthly run artifacts.
- Keeps UI concerns separated from server-side processing.

# Notes
- MVP: implement only essential viz features first; expand iteratively.
